This project demonstrates how to configure a DHCP server within a router using Cisco Packet Tracer. DHCP (Dynamic Host Configuration Protocol) automatically assigns IP addresses to devices in a network, simplifying network management. This setup eliminates the need to manually configure IPs for each client device.

âœ¨ Key Components

  • Router: Acts as the DHCP server.
  • Switch: Connects all devices in the LAN.
  • End Devices: Includes 2 laptops and 3 PCs as DHCP clients.
  • Subnet: 192.168.0.0/16 (Subnet mask: 255.255.0.0)
  • Gateway: 192.168.0.1

ðŸ”§ Configuration Steps

  1. Assign a static IP address to the router interface (e.g., 192.168.0.1).
  2. Enable the DHCP service on the router.
  3. Create a DHCP pool with:
    • network 192.168.0.0 255.255.0.0
    • default-router 192.168.0.1
    • dns-server 8.8.8.8 (optional)
  4. Set end devices (PCs/laptops) to use DHCP in their IP configuration.

ðŸ“¡ What Happens in the Network

Once configured, each device sends a DHCP Discover message. The router responds with an IP lease, giving each device a unique IP address along with subnet and gateway info. This allows seamless communication between devices.

ðŸ§ª Test Procedure

  1. Use the IP Configuration tab on each PC to confirm they received an IP via DHCP.
  2. Open the Command Prompt on a PC and use ping to test connectivity with another PC.
  3. Try pinging the default gateway 192.168.0.1 to ensure router communication.

âœ… Outcome

All devices successfully obtain IP addresses automatically from the router. Network communication is verified through successful pings, confirming the DHCP setup is functional.
